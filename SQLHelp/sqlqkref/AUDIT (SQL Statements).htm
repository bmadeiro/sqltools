AUDIT (SQL Statements)

Use this command to choose specific SQL statements for auditing in subsequent user sessions. To choose particular schema objects for auditing, use the AUDIT command (Schema Objects). 

 AUDIT {statement_opt | system_priv}
   [, {statement_opt | system_priv} ] ...
   [BY user [, user] ...]
   [BY {SESSION | ACCESS} ]
   [WHENEVER [NOT] SUCCESSFUL]

See also: AUDIT (Schema Objects) , NOAUDIT (Schema Objects)
