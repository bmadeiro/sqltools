Autosave

Any program can crash including an operation system, finally SQLTools has autosave and restore to prevent editor data loss.

The program automatically saves your open files at certain intervals. The default is 10 seconds. You can disable autosave by setting the interval to 0.

Automatic restore happens if a new starting instance of the program finds a sign of either crash or abnormal termination. If you had 3 running instances of  SQLTools on system reboot/crash then you need to launch 3 new instances in order to restore all 3 sessions.

Autosave uses a Snapshot stored in Backup folder. You can back and open any historical autosave files if the checkbox "Delete snapshot on normal exit" is OFF.

The default location for Autosave & Backup folder is under Setting Folder.
