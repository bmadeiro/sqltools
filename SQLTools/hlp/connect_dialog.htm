Connect dialog

The dialog allows you to create a new connection or open one from saved connections.

Connection List

You can select a connection to open from the connection list. Press on the header bar for sorting by any column. The right mouse button activates a context menu. A few command are available only tin the context menu: "Cleanup up", "CopyUsername@Alias", "CopyUsername/Password@Alias" and "Make and Copy Alias"

Column filters can help with navigation thru multiple stored connections.

TAG

This is an optional field that can be used to organize stored connections.

You can use the following notation: application_abbreviation.environment, for example, sales.dev1, sales.test, sales.prod

User / Password

Specify the user / password for Oracle account.

TNS

Specifies the TNS alias from TNSNAMES.ORA.

Bypass tnsnames.ora

Allows to format a connection string from Host, TCP Port and SID instead of look up this information from TNSNAMES.ORA. It can be useful if you need to look a new instance quickly.

Host / TCP Port / SID

Specify an instance location when Bypass box is checked.

Mode

Usually it is Normal but SYS_DBA or SYS_OPER can be chosen for special accounts.

Read Only

SQLTools can enforce Read-Only access for a specified connection.

Connect

Click this button to create a new session and close the dialog box. All connection attributes will be automatically saved.

Save

It saves all changes w/o making a new session.

Close

Click this button to open a connection and close the dialog box..

Test

Click this button to test connection with the specified connection attributes.

Delete

Click this button to delete the current connection from Connection List.

Settings ...

Click this button to show Connect dialog settings.
