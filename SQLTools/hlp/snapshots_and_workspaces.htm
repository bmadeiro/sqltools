Snapshots and Workspaces (text editor sessions)

Snapshot is a momentary copy of editor session with all files and unsaved modifications. You can make a snapshot at any moment including when you are closing the program and re-open it later. A shapshot contains the list of open files with cursor/selection/bookmark positions and also the text of modified files.

When to use snapshots: very often I have a bunch open of windows and have no desire to save all of them in individual files. It would be nice to have backup for them but in one single file so snapshot is an ideal solution for that.

Snapshot has simplified save/open interface. You just use the command "Snapshot->Quick Save" and the program generates the snapshot name and creates it in the Backup folder, you can open it using Quick Snapshot recent menu. The snapshot will disappear from the menu after opening however will be still available in the Backup folder.

Copy and Paste can be useful for quick cloning your working session and also moving from old to new version of the program.

Autosave snapshot is a backup solution. You can open it from the Backup folder using "Snapshot->Open AS / QS..."

Workspace is extended version of a snapshot. It can be open only by a single instance of the program and serves to save the state of this instance.

When to use workspaces: they suit for prolonged work on some feature or activity.

The toolbar has 4 buttons designated to open and save a workspace or a snapshot. Additionally recent workspace/snapshot dropdown menu can be activated by clicking on the arrow-down of those buttons.
