What's new in 1.8

All database operations have been moved into background thread

SQLTools is essentially multi-threaded application with all database calls moved in the background. That allow non-blocking behavior and quicker response time for some operation like open connection.

Autosave

Any program can crash including an operation system, finally SQLTools has autosave and restore to prevent editor data loss.

Snapshots and Workspaces (text editor sessions)

Snapshot is a momentary copy of editor session with all files and unsaved modifications. You can make a snapshot at any moment including when you are closing the program and re-open it later.

Workspace is extended version of a snapshot. It can be open only by a single instance of the program and serves to save the state of this instance.

Application launcher

There are a situations when you cannot modify your environment variables like PATH or ORACLE_HOME but that might be necessary in order to make SQLTools work properly.
